DTCs

DTCs are diagnostic trouble codes. These DTCs are usually set by a controller when a fault occurs due to a parameter being out of range, sensor being disconnected, etc. DTCs are requested from the controller and timestamped by the addon. The most recent codes are kept in the DTC channels.

What do you want to do?

Set the DTC formatting

The DTC Byte Length, Status Byte Length, Failure Byte Length (or Additional Data Byte Length) and Byte Order are needed to be able to convert the DTCs to fault codes.

UDS Defaults

  • DTC Byte Length: 3
  • Status Byte Length:1
  • Failure Byte Length: 0 or 2

KWP2000 & GMLAN Defaults

  • DTC Byte Length: 2
  • Status Byte Length:1
  • Failure Byte Length: 0

Set the DTC Codes and Status Codes

The user can import the DTC codes and status codes. This is done through a comma seperated value (CSV) file. For DTC codes, the format is numeric code (first column), J2012 code (second column) and description string (third column) in a CSV file. The numeric code can be a number or a hex value. Use a "x" to designate hex. E.g xABCD. If you use a number, it will be converted to hex on import. Each row the CSV file is a code/J2012 code/string pairing. The user selects the DTC Codes tab and hits Import. Select the CSV file and the table will update. The same approach applies for the Status Codes tab. The CSV format is slightly different. It is numeric code (first column) and description string (second column)

Set the number of recent DTCs

By default, the user can see the most recent 10 DTCs received. If the user wants a different buffer length, they can set Number of DTCs to the desired value and hit Apply.

Monitor the total number of DTCs on an ECU

The # of DTCs channel allows the user to see the total number of DTCs received.

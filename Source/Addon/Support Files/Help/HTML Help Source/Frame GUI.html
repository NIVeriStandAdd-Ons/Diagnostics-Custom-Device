Configure Service and Signals

Some services have parameters and data. For example, the Read DID (data identifier) service, has a paramter "ID" that defines which DID to read. The data is then the signal values associated with that "ID". This section sets the service's ID/address and signals/data.

What do you want to do?

Set ID/Address Name

Enter the ID/Address's name in Name. This must be unique for a given service type. Not all services require this.

Set ID/Address

Enter the value in ID/Address. This must be unique for a given service type. Not all services require this.

Set Data

To add a signal, select the Add button. Fields will appear that relate to the table columns.

  • Signal Name: The name of signal. The name must be unique within the service data, start with an alphabetical character, and not use special characters or spaces.
  • Start Bit: The start location of the signal within the service data (not the location within the raw byte stream). It is subject to Endian. It follows the rules used in the NI-XNET Database Editor. E.g. If the first signal is 16 bits, is the first item in the service data, and little endian, the start bit will be 0. If big endian, the start bit will be 8.
  • Length: Length of the signal in bits. The maximum supported length is 64 bits.
  • Factor: The multiplication factor to convert the signal binary value to its physical value.
  • Offset: The offset to add to the signal scaled physical value.
  • Minimum: The smallest possible signal value.
  • Maximum: The largest possible signal value.
  • Type: Signed, Unsigned, or Float.
  • Endian: The ordering of bytes for this signal. Little or Big.
  • Default: The value read or written for the signal if no physical data are yet available
  • Units: The signal physical unit

Edit Data

Single click on any existing signal column field and type in the edits. Make sure the formats, spellings, and capitalizations are correct.

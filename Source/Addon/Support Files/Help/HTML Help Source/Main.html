Diagnostics

The diagnostic addon is used to communicate with an ECU over a diagnostic protocol. Supported protocols are KWP, UDS, & GMLAN. CAN is the supported physical layer.

A quick start document provides an overview of the addon capabilities. It is located at:

"C:\Users\Public\Documents\National Instruments\NI VeriStand (Year)\Custom Devices\Diagnostics Addon\Windows\Quick Start Documentation\Diagnostics Addon Quick Start.html"

Most of the addon sections contain additional built-in documentation.

The addon reads DIDs, PIDs (GMLAN only), memory (KWP/UDS only) and DTCs. It writes DIDs, CPIDs (GMLAN only), memory (KWP/UDS only) and clears DTCs. KWP/UDS also supports IO control, routines (start, stop, status), and custom (user defiend) messages.

The complete list of supported services are as follows:

UDS

  • Tester Present (x3E)
  • Diagnostic Session Control (x10)
  • Communication Control (x28)
  • ECU Reset (x11)
  • Request Seed/Send Key (x27)
  • Read Data by Identifier (x22)
  • Write Data by Identifier (x2E)
  • Read Memory by Address (x23)
  • Write Memory by Address (x3D)
  • Input Output Control By Identifier (x2F)
  • Routine Control (x31)
  • Control DTC Setting (x85)
  • Report Severity Information of DTC (x19 with x9 option)
  • Report DTC by Status Mask (x19 with x2 option)
  • Clear Diagnostic Information (x14)

KWP

  • Tester Present (x3E)
  • Start Diagnostic Session (x10)
  • Stop Diagnostic Session (x20)
  • Disable Normal Message Transmission (x28)
  • Enable Normal Message Transmission (x29)
  • ECU Reset (x11)
  • Request Seed/Send Key (x27)
  • Read Data by Local Identifier (x21)
  • Write Data by Local Identifier (x3B)
  • Read Memory by Address (x23)
  • Write Memory by Address (x3D)
  • Input Output Control By Local Identifier (x30)
  • Start Routine by Local Identifier (x31)
  • Stop Routine by Local Identifier (x32)
  • Request Routine Results by Local Identifier (x33)
  • Control DTC Setting (x85)
  • Read Status of DTC (x17)
  • Read DTC by Status (x18)
  • Clear Diagnostic Information (x14)

GMLAN Only

  • Tester Present (x3E)
  • Initiate Diagnostic Operation (x10)
  • Disable Normal Communication (x28)
  • Return to Normal Mode (x20)
  • Request Seed/Send Key (x27)
  • Read Data by Identifier (x1A)
  • Write Data by Identifier (x3B)
  • Read Data By Parameter Identifier(x22)
  • Device Control (xAE)
  • Dynamically Define Message (x2C)
  • Read Data By Packet Identifier (xAA)
  • Read Status Of DTC By DTC Number (xA9 with x80 option)
  • Read DTCs by Status (xA9 with x81 option)
  • Send On Change DTC Count (xA9 with x82 option)
  • Clear Diagnostic Information (x4)

What do you want to do?

Disable Add-on

To disable this add-on, right click on the add-on instance and select Disable from the shortcut menu.

Export the Current Configuration

The custom device configuration can be exported to a file. This allows the user to re-add the custom device later without having to manually re-build the configuration. To get this configuration file, right click on the custom device and select Export Config.

Delete item

To delete, right-click the object in the configuration tree and select Delete from the shortcut menu.

Delete multiple items

To delete multiple items, right-click the object in the configuration tree and select Delete Multiple Items from the shortcut menu.
